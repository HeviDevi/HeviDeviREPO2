HTML

    It is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • In addition to elements that can be seen on the page, elements are also used to divide the content on the page and to describe the content to browsers and other developers.

--HTML Element Guide--

  • At the very top, we see the < !DOCTYPE html > declaration. This declaration tells the browser what document type it should expect when it is opened. This particular declaration is letting the browser know to expect HTML on the page.

  • The < head > element contains information, or metadata, that helps the browser to render the page correctly. Elements included within the < head > are not visible when a user looks at a webpage

  • The < body > element contains the content that is shown to the user. All the elements that we want the user to see and interact with must be placed inside this element.

  • Elements that are nested inside other elements are called child elements. If there are three elements nested inside of a body element, those three nested elements are known as child elements. The element that holds all the children is known as the parent element. This “parent/child” language is often used by developers to explain the relationship among elements in HTML.

  • The < h1 > h2 > < h3 > < h4 > < h5 > and < h6 > elements represent the level of heading a given text block represents. Headings are exactly what they sound like: larger or more prominent titles and subtitles to organize a page.

  • The p element represents a paragraph or block of text. You'll use this element extensively to wrap most of the text on your webpages.

  • The < ul > < ol > and < li >elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.

  • The < img > element contains information about images that are displayed on the webpage. The img element can contain various attributes; the src attribute is the most important because it defines the location of the image.

  • The < br > element creates a line of empty space, or a line break, between two blocks of content.

  • the < a > element (which stands for anchor) creates links to the same webpage or other webpages. The href attribute points to the URL for the link.

  • a < link > element creates the connection to the external file

CSS

Cascading Style Sheets (CSS), is a programming language used by developers to define how webpage content is presented to users. CSS defines the colors, aesthetics, visual layout, and organization. We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.


When a developer wants to style an element directly in the HTML file, they use inline CSS. Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. They’re useful for quick and permanent changes but are less flexible than external and internal style sheets, because each inline style must be edited individually whenever you make a design change.


Inline CSS and internal style sheets are totally acceptable methods under certain conditions, but the most common way to style a webpage is by using an external CSS style sheet. This is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.

--CSS Properties Guide--


  • The margin property acts much like a margin in a word processing document. A margin property indicates how much space we want around the outside of an element. The margin is not visible on the page; it merely adds a buffer. When we set the margin property to 0, we are indicating that we want NO extra space around the element.

  • The padding property is often confused withmargin, but it is not the same thing. The padding property adds space around the content inside an element. When you increase padding, you are actually making the entire element bigger! That is because the padding, while transparent, is located inside the element, whereas the margin is the transparent buffer outside an element.

  • Every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content.

Git

    Git is a version control system that lets you manage and keep track of each version of your project. To access Git, we use the command line.


    GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely. Using GitHub not only allows us to store our code, but also to easily collaborate with others.


  • Repository
    A folder that holds all files, and their version histories, relating to a specific project.

  • Fork
    A new repository that inherits from a parent “upstream” repo. It is used to suggest changes to an “upstream” public repo.

  • Branch
    A workspace that allows you to make changes to a projet that don't affect the live site.

  • Markdown
    The file format for Github. A way to write for the web which easily converts to HTML. Markdown files end in .md

  • Pull Request or PR
    A request to apply the cahnges made to your branch onto another branch. Discuss and PRs with peers.

  • Merge
    When a Pull Request is apporved and 'pulled in' to another branch.

  • Issue
    A tool used to report a new task or fixes that have been applied. Any issues added are tracked on the "project board". Be sure to label the issue and add an attachment to the platform that the work is on. Close issues once addressed.

    • --Git Commands--

  • git status: checks what branch we are currently on

  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript


JavaScript is a powerful programming language, invented by Brendan Eich, that can add interactivity to a website.


Control flow is the order in which a computer executes code in a script.

--Java Variable Types--

A variable is a named container that allows us to store data in our code


  • String data is a sequence of text enclosed in single or double quotes

    i.e. let myVariable = 'Bob'; OR let myVariable = "Bob";

  • Number Variables do not have quotes around them

    i.e. let myVariable = 10;

  • Boolean variables are True/False value

    i.e. let myVariable = true;

  • Array variables are structures that allow you to store multiple values in a single reference

    i.e. let myVariable = [1,'Bob','Steve',10];
    Refer to each member of the array like this: myVariable[0], myVariable[1], etc.

  • Object variables can be anything. ecerything in Javascript is an object and can be stored in a variable
    i.e. let myVariable = document.querySelector('h1');

README


A README file acts like a virtual storefront to a repository—it's the first thing that a person sees when they visit a repo on GitHub. But it's also much more than that: README files contain essential information about the repo's project. Thus, the quality of a README file can differentiate a high-quality repo from a low-quality one.


Your README might also describe some of the challenges you faced, as well as the features you plan to implement in the future.


If your project is deployed, make sure to include a link to the deployed application so people can see it in action!


--What Should A README Contain--

  • What was your motivation?

  • Why did you build this project?

  • What problem does it solve?

  • What did you learn?

  • What makes your project stand out?
